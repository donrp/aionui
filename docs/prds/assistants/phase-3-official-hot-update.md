# 三期 · 官方资产热更

> 本期把官方维护的资产——内置助手与官方技能——从"随应用版本更新"升级为"不发版即可更新"，让官方的优化能更快惠及所有用户。
>
> 概念、术语、整体路线见[总纲](./README.md)。

---

## 一、要解决的问题

官方持续在打磨内置助手（如 Cowork）的规则、技能效果。但目前这些官方资产是**随应用版本走**的——只读，且只能跟着客户端发版才更新。这带来两个问题：

- **更新慢**：官方对一个内置助手规则的优化，要等下一个应用版本、用户升级后才能用上。一个小的提示词调整也得搭一次发版。
- **迭代受限**：官方想快速试验、修正某个内置助手或技能的效果，没有不发版就下发的通道。

> 现状参照：内置助手在编辑界面只读、靠"创建副本"派生（见一期 F1-07）；其内容随应用版本覆盖更新。官方技能（随应用附带的技能）同理。

## 二、目标

- 官方对**内置助手**和**官方技能**的更新，能在不发布新应用版本的情况下下发给用户。
- 用户无需手动操作，即可用上官方的最新优化。
- 不破坏一期确立的边界：内置助手对用户仍是只读模板，用户的自定义助手（含从内置派生的副本）是自己的资产，**不被热更覆盖**。

## 三、本期不做（Non-Goals）

| 不做的事 | 说明 |
| --- | --- |
| 改变内置助手的只读 / 派生规则 | 沿用一期 F1-07：用户仍通过"创建副本"来自定义 |
| 热更用户自己的助手 / 自定义技能 | 热更只针对官方资产；用户资产始终归用户，不被覆盖 |
| Agent、首页、团队相关 | 已在一、二期完成 |

## 四、功能需求

### F3-01 内置助手热更

**需求**：内置助手的官方内容（名称、头像、描述、规则、默认技能/MCP、推荐提示词等只读字段）支持**不发版更新**——官方侧发布新版本后，用户端在无需升级应用、无需手动操作的情况下收到并应用。

- 更新只覆盖**内置助手本身**的只读内容。
- 用户从某个内置助手**派生出的副本**是用户自定义助手，**不受热更影响**，保持用户自己的修改。
- 用户正在进行的对话不受影响（对话用的是创建时的配置快照，见总纲「字段生效方式」）；更新对**新发起的对话**生效。

**验收**：
- [ ] 内置助手的官方内容可不发版更新，用户无需手动操作即可用上
- [ ] 派生副本不被热更覆盖
- [ ] 进行中的对话不受影响，更新对新对话生效

---

### F3-02 官方技能热更

**需求**：官方随应用附带的技能，其内容与效果支持**不发版更新**——官方优化后，用户端无需升级应用即可用上最新版本的官方技能。

- 热更只针对**官方技能**；用户自己从技能中心导入的自定义技能、第三方扩展技能不在此列，不被覆盖。
- 已启用某官方技能的助手，在该技能更新后，新发起的对话使用更新后的版本。

**验收**：
- [ ] 官方技能可不发版更新，用户无需手动操作即可用上
- [ ] 用户自定义技能、扩展技能不被热更覆盖
- [ ] 更新对新对话生效

## 五、衔接说明

- 本期建立在一期 F1-07（内置助手作为只读模板 + 派生）之上：只读内容的"来源"从"随应用版本"切换为"可热更下发"，只读 + 派生的产品规则不变。
- 二期已让内置助手与其他助手在首页、团队中统一呈现；本期不改变这些呈现，只改变其内容的更新方式。

## 六、成功标准

- 官方对内置助手、官方技能的优化，能在不发版的前提下让用户用上。
- 用户的自定义助手与自定义技能在热更过程中始终不被覆盖。
