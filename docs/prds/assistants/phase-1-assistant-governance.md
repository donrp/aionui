# 一期 · 助手治理

> 本期把"助手"从一个只能配名称/规则/技能的轻量预设，扩展成能承载完整默认配置的对象。**纯配置层改造**——不改 Agent 页、不引入裸助手、不改首页。
>
> 概念、术语、整体路线见[总纲](./README.md)。

---

## 一、要解决的问题

今天用户在「设置 → 助手」里能配的东西很有限：名称、头像、描述、主 Agent、规则、技能。但一个助手真正发起对话时还需要的东西——**用哪个模型、什么权限、带哪些 MCP、有没有起手提示词**——目前要么没地方配，要么得每次开对话临时选。

结果是：助手"配不厚"，用户每开一个新对话都要把模型、权限重选一遍，体验割裂。

## 二、目标

- 助手能预设**默认模型、默认权限、默认技能、默认 MCP、推荐提示词**，新建对话时自动带上。
- 把这些配置收拢到**一个从容、清晰的助手编辑界面**里，用户有一个合理的地方一次配好。
- 内置助手作为官方模板可被"另存为自己的助手"再改。

## 三、本期不做（Non-Goals）

| 不做的事 | 留给 |
| --- | --- |
| Agent 页改造、连接检测、测试连接 | 二期 |
| 裸助手（Agent 自动生成助手） | 二期 |
| 首页改造（pill 栏、取消二级页等） | 二期 |
| 内置助手 / 官方技能的热更 | 三期（本期内置助手仍随应用版本更新） |

## 四、功能需求

### F1-01 助手编辑界面改为全屏分区页面

**现状**：编辑助手是一个右侧滑出的抽屉（宽度约为窗口一半）。字段一多就拥挤，规则、技能等都挤在窄抽屉里。

**需求**：把编辑界面改为一个**可返回的全屏页面**（顶部「← 返回」+ 保存/取消），内容按卡片分区，给新增的默认配置留出空间。

**分区顺序**（从上到下）：

1. **身份**：头像、名称、描述 —— *改动立即生效*
2. **推荐提示词**：起手提示词列表（见 F1-05）—— *改动立即生效*
3. **引擎**：主 Agent —— 决定下方默认模型/权限的可选范围
4. **默认配置**：默认模型、默认权限、默认技能、默认 MCP（见 F1-03 ~ F1-04）—— *仅对新会话生效*
5. **规则**：Markdown 规则，编辑/预览切换 —— *仅对新会话生效*

> "立即生效"与"仅对新会话生效"的含义见总纲「助手字段的两种生效方式」。把立即生效的身份、推荐提示词排在靠上，是为了让"改了马上能看到"的内容更直观。

**入口**：在「设置 → 助手」列表中，点击某个助手或「创建助手」进入本页。

**验收**：
- [ ] 助手编辑为可返回的全屏页面，按上述 5 个分区组织
- [ ] 主 Agent 变化时，默认模型/权限的可选范围相应更新

### F1-02 助手配置 schema 扩充

**需求**：助手在原有字段基础上，新增以下可配置项：

| 配置项 | 说明 | 生效方式 |
| --- | --- | --- |
| 默认模型 | 该助手新建对话默认用的模型 | 仅新会话生效 |
| 默认权限 | 该助手新建对话默认的权限模式 | 仅新会话生效 |
| 默认技能 | 该助手默认启用的技能集合 | 仅新会话生效 |
| 默认 MCP | 该助手默认启用的 MCP 服务集合 | 仅新会话生效 |
| 推荐提示词 | 选中助手后展示的起手提示词 | 立即生效 |

裸助手、内置助手、自定义助手共用这同一套配置结构（裸助手在二期引入，但结构本期就定好）。

**验收**：
- [ ] 助手数据结构包含上述全部配置项
- [ ] 自定义助手在编辑页可设置以上全部项

---

### F1-03 默认模型 / 默认权限

**需求**：编辑页的「默认配置」区，提供默认模型、默认权限两个下拉。每一项可处在总纲定义的两种状态之一：

- **暂不设置（自动记住上次）**：下拉**首项**，新建助手时默认选中。附一行浅色说明：未设置时，以你最近一次用这个助手时选择的值作为下次默认。
- **已设定固定值**：首项之后列出该主 Agent 支持的模型 / 权限模式，选定后该助手新建对话即用它，对话中临时改动不影响下次。

可选的模型与权限模式，由助手当前的主 Agent 决定。

**"记住上次"按助手隔离**：选「暂不设置」时，用户在对话中切换的模型/权限，记为**该助手自己**的偏好。每个助手各记各的——哪怕两个助手用同一个 Agent，也互不影响。

**验收**：
- [ ] 默认模型、默认权限均为下拉，首项为「暂不设置（自动记住上次）」并附说明
- [ ] 选「已设定固定值」时，该助手新建对话默认使用该值，对话中临时改动不回写
- [ ] 选「暂不设置」时，对话中切换的模型/权限被记为**该助手自己**的偏好，下次新建对话沿用
- [ ] 不同助手的"记住上次"互相独立，即使用同一个 Agent 也不串
- [ ] 可选项范围由当前主 Agent 决定

---

### F1-04 默认技能 / 默认 MCP

**需求**：在编辑页「默认配置」区，默认技能、默认 MCP **各用一个单一的下拉列表**（带搜索、可多选勾选）：

- **不向用户暴露分类**（自动注入 / 内置 / 自定义 / 扩展等内部区分不展示），直接列出可选项让用户勾选。
- 下拉所在标题的**右侧**放一个跳转入口：默认技能 → 「去技能中心」（`设置 → Capabilities → 技能`）；默认 MCP → 「去 MCP 设置」（`设置 → Capabilities → 工具`），方便用户去导入/管理这些资源本身。
- 系统的核心能力（如文件操作）**默认已勾选**，用户可针对该助手取消；其余默认不勾选，按需启用。这层默认差异只体现为"某些项默认勾上"，不向用户解释背后来源。

**两种状态**（与 F1-03 一致）：默认技能、默认 MCP 同样各有「暂不设置（自动记住上次）」与「已设定固定列表」两种状态。

- **已设定固定列表**：每次新建对话默认勾上这批，用户在正式发起前还能临时增减（这次的增减不回写）。
- **暂不设置（自动记住上次）**：新建对话时还原该助手上次新建对话时启用的那批技能 / MCP。

技能 / MCP 在对话创建那一刻固化，对话开始后不能再增减。"记住上次"同样按助手分别记。

**验收**：
- [ ] 默认技能、默认 MCP 各为单一下拉多选，不向用户分类或解释来源
- [ ] 两个下拉的标题右侧分别有「去技能中心」「去 MCP 设置」入口
- [ ] 核心能力默认勾选、可取消；其余默认不勾选
- [ ] 支持「暂不设置（自动记住上次）」与「已设定固定列表」两种状态，按助手分别记
- [ ] 已设定列表时新建对话默认勾上、发起前可临时增减；技能/MCP 在对话开始后不可再增减

---

### F1-05 推荐提示词

用户在首页选中一个助手后，输入框下方会展示该助手的推荐提示词，点击即填充到输入框，方便快速起手。但这些提示词目前在助手编辑界面没有配置入口。

**需求**：在编辑页提供推荐提示词的编辑入口——用户可对列表**增、删、改**（建议 3 条）。属于立即生效内容，保存即更新；选中该助手时，首页展示的就是最新的推荐提示词。

**验收**：
- [ ] 编辑页可对推荐提示词增、删、改
- [ ] 保存后，首页选中该助手时展示的推荐提示词即为最新内容

---

### F1-06 助手就地编辑与字段生效规则

**需求**：用户编辑自己的助手（自定义助手）并保存后：

- 助手内部 ID 保持不变。
- **立即生效**类字段（名称、头像、描述、推荐提示词）的改动，马上在所有引用处刷新，包括历史对话里显示的名字与头像。
- **仅对新会话生效**类字段（规则、默认模型/权限/技能/MCP）的改动，只影响之后新建的对话，不改变进行中或历史的对话。

**验收**：
- [ ] 编辑助手后内部 ID 不变
- [ ] 名称/头像/描述/推荐提示词改动后，历史对话显示同步刷新
- [ ] 规则/默认值改动后仅新会话生效，进行中与历史对话不受影响

---

### F1-07 内置助手：只读模板 + 创建副本

**现状**：内置助手在编辑抽屉里大部分字段只读（仅主 Agent 可改），顶部有提示横幅与"复制"入口。

**需求**：沿用并明确这一行为，适配到新的全屏编辑页：

- 打开内置助手编辑页时，**顶部固定一条说明**：这是官方助手、随版本更新，除引擎与默认模型/权限外均为只读；如需自定义其余内容，请创建副本。
- **可改**：主 Agent 及其默认模型、默认权限。
- **只读**：名称、头像、描述、规则、技能、MCP、推荐提示词。**只读不等于隐藏**——这些字段照常展示当前配置值，让用户看得到这个官方助手是怎么配的，只是不可编辑。
- 提供「基于此助手创建副本」入口 → 生成一个用户自己的自定义助手，全部字段可编辑。

**本期范围**：内置助手内容本期仍随应用版本更新（即随版本覆盖），不发版即更新的"热更"留到三期。

**验收**：
- [ ] 内置助手编辑页顶部固定官方助手说明
- [ ] 仅主 Agent 及默认模型/权限可改，其余只读且照常显示当前值
- [ ] 提供「创建副本」入口，副本为可全字段编辑的自定义助手

---

### F1-08 助手开关

**需求**：每个助手有一个启用/禁用开关，默认开启。关闭后，该助手不出现在"发起对话"的候选里（首页助手列表）；但不影响它已经服务的历史对话，那些对话继续正常工作。

**验收**：
- [ ] 助手开关默认开启、可切换、可持久化
- [ ] 关闭后该助手从首页助手列表中移除
- [ ] 关闭不影响该助手已存在的对话

---

### F1-09 助手排序

**需求**：用户可在「设置 → 助手」列表中**拖拽调整助手顺序**。该顺序即时影响首页下方助手列表的排列——常用的助手排在前面，更快找到、点到。

**验收**：
- [ ] 用户可在助手列表拖拽排序
- [ ] 排序结果即时反映到首页助手列表的排列

---

### F1-10 数据迁移

**需求**：现有助手数据需平滑迁移到扩充后的配置 schema——已有助手的名称、规则、技能等保持不变，新增的默认模型/权限/技能/MCP/推荐提示词字段取合理初始值（默认模型/权限为「暂不设置」，默认技能保持现有启用集合，默认 MCP 为空，推荐提示词为空或沿用既有数据）。迁移对用户无感，不丢配置。

**验收**：
- [ ] 升级后，已有助手的原有配置完整保留
- [ ] 新增字段取合理初始值，用户无需手动修复

## 五、衔接说明

本期为后续两期打的地基：

- **配置 schema 与默认值的完整两态行为**（含"记住上次"）在本期定稿。二期引入的裸助手直接复用这套结构与行为，无需额外开发（F1-02 ~ F1-04）。
- **开关、排序** 的能力在本期就绪，并已作用于首页现有的助手列表区域；二期首页改造为助手 pill 栏后，它们的控制效果延续到新的 pill 栏（F1-08、F1-09）。
- **内置助手只读模板** 在本期成形；三期把"随版本更新"升级为"热更"（F1-07）。

## 六、成功标准

- 用户能在一个界面里为助手配好默认模型、默认权限、默认技能、默认 MCP、推荐提示词，并在新建对话时看到这些默认值生效。
- 现有助手升级后配置无损。
- 内置助手可被一键派生为可编辑的自定义助手。
