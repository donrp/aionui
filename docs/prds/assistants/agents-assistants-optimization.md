# PRD：Agents 与助手列表体验优化

> 术语：**CLI 助手** = 由某个 CLI Agent 自动生成、与之一对一绑定的助手（如 Claude Code、Aion CLI、Codex CLI）。代码里这类助手的来源标记为 `bare`。

---

## 需求 1：Agents 页加状态筛选 Tab

**改动原因**
有用户反馈希望「可用的 Agent 排在前面」。但如果按状态排序，Agent 的位置会随状态变化而跳动——比如检查更新后某个 Agent 变为可用，它会突然换位置，反而更难找。

**期望效果**
列表上方增加三个 Tab：**全部 / 可用 / 不可用**（未安装、需登录、离线等都归入「不可用」）。

- 每个 Tab 只做状态筛选，**不改变 Agent 在列表中的相对位置**。
- Tab 标题带数量，如「可用 (5)」。
- 切换 Tab 时，符合条件的 Agent 原位保留、其余隐藏，状态变化不再导致位置跳动。

---

## 需求 2：助手列表按钮收纳

**改动原因**
助手每行的操作按钮（编辑、删除 / 复制新建）平铺在右侧，过于杂乱。

**期望效果**
每行只保留**启用开关**，其余按钮全部收进一个 **「…」更多菜单**。

- 点「…」展开：编辑、复制新建 / 删除。
- 按钮的显隐规则不变（内置助手显示「复制新建」、自定义助手显示「删除」）。

---

## 需求 3：CLI 助手支持编辑

> 此需求涉及 AionCore 后端，由研发负责实现。本节描述产品期望。

**改动原因**
CLI 助手目前是**完全只读**的，用户无法对它做任何个性化配置。但实际上除了身份信息（名字、头像、对应的 CLI），其余配置用户应当可以自定义。

**期望效果**

1. **列表挂「CLI」标签**——和内置助手挂「内置」标签一样，让用户一眼区分这是 CLI 自动生成的助手。

2. **详情页顶部加说明**——类似内置助手的提示条，说明：

   > 该助手由 Agents 自动生成，与对应的 CLI 一对一关联。名字、头像、主代理不可修改，其余配置均可编辑。

3. **放开编辑权限**：
   - **不可改**：名字、头像、主代理（这三项由 Agent 决定，是助手的身份）。
   - **可编辑**：描述、提示词、规则、技能、MCP、默认模型、默认权限等。

4. **编辑结果跟随本地、不被覆盖**——用户改完的配置保存在本地，App 更新时**不会被覆盖**。
   （注：只有官方内置助手才有「更新时用官方配置覆盖」的逻辑；CLI 助手和用户自建助手一样，以用户本地配置为准。）
